Technical Research Report

LLM 应用层扩展演进全景图

从早期的文本模拟到现代的标准化能力封装，探索 LLM 如何通过扩展层解决原生缺陷，构建 AI 代理经济。

知识幻觉与时效性

模型训练数据的截止日期导致其无法感知实时世界，且容易生成虚假事实。

确定性逻辑缺失

LLM 擅长概率预测而非精确计算。在处理数学、代码执行等任务时缺乏稳定性。

孤岛效应

工具分散且接口不一，模型难以跨平台、跨应用地自主调度外部资源。

演进时间线

2022 - 2023初

胶水代码时代

通过 Prompt Engineering (ReAct 范式) 强制模型输出特定文本格式。开发者编写大量的 Regex 或 Python 代码来解析输出并手动触发工具。

Key: Chains, Manual Parsers, ReAct Prompt
Key: JSON Schema, Tool Definition, Native FC
2023.06 - 2024

原生 Function Calling

OpenAI 推出微调后的模型，使其原生理解结构化定义。解决了格式解析不稳的问题，使“工具调用”成为模型的一种标准化能力。

当前最火标准

Skills 技能封装

不再只是简单的 API 调用，而是包含语义 Metadata、上下文感知与执行策略的“智慧组件”。如 Semantic Kernel 中的 Plugins 或 GPTs 的 Actions。

核心：能力即技能

解决了工具的“发现”与“决策”难题。模型不仅知道“怎么调”，更通过 Skills 定义的描述知道“什么时候该调”。

Key: MCP (Anthropic), UCP, Agent Economy
2024末 - 未来

标准化协议 (MCP/UCP)

模型上下文协议 (MCP) 试图统一所有数据源接口。UCP (通用商业协议) 解决 Agent 自主支付与信任问题，开启 AI 代理经济。

应用场景：LLM 附加了什么能力？

Web Search

附加能力

Query Rewrite & RAG

通过搜索工具补全离线知识。LLM 负责将模糊的用户意图重写为多维搜索词，并对搜索结果进行摘要与信源对齐。

Code Runner

附加能力

Sandbox Execution

解决复杂逻辑计算。LLM 生成代码，在外挂的 Python 沙箱中执行，将确定性结果返回，避免大模型“数不清 9.11 和 9.9 谁大”。

Workflow Agent

附加能力

Skill-based Orchestration

针对复杂办公任务。通过集成 Skills 库，Agent 可以自主决策：是先查邮件、还是先分析 Excel，从而实现长程任务自动化。

总结：为何 Skills 是当下的重心？

工具（Tools）是死板的接口，而技能（Skills）是带有语义说明的逻辑块。 在 Agent 时代，大模型更像是一个指挥官，它需要通过阅读 Skills 的 Metadata 来决定如何组合不同的专家能力，这标志着从“程序集成”转向了“智力对齐”。

进化逻辑总结
  • Step 1: 文本解析（极其不稳定，仅能处理简单任务）
  • Step 2: 结构化调用（稳定工具，但缺乏语义理解）
  • Step 3: 技能化标准（赋予工具“被理解”的能力）
  • Step 4: 协议化互联（跨模型的全球 Agent 生态）
