Unified Tool Orchestration

一切皆工具：外卖智能体原理

本质上，自动点外卖与写代码、做研究一样，是将 API 封装为 Tools。 Agent 的任务是根据意图，在正确的时间、以正确的参数调度这些工具。

标准化实现架构

规划逻辑 (Agentic Core)

通过 LLM 进行语义理解，决定调用的先后顺序。如果 API 报错，则进入“反思-重试”逻辑。

上下文记忆 (Memory)

存储 Token、历史偏好、已领取的优惠券状态，确保跨 API 调用时的信息连贯性。

工具契约 (Tool Contract)

将平台 API、地址服务、支付接口统一描述为 JSON Schema，供 Agent 识别并填充参数。

智能调度：将 API 转化为 Tools

Agent 不关心 API 后面是什么，只关心如何调用它们来完成目标

GET /location

地址感知 Tool

获取经纬度与当前配送地址

POST /platform/search

外卖平台 Tool

跨平台检索商户、菜品、满减信息

Agent Orchestrator
POST /payment/verify

支付 API Tool

拉起生物识别或第三方安全支付

GET /delivery/track

状态跟踪 Tool

实时获取配送员位置与 ETA

递归执行案例 (ReAct)

[THOUGHT] 需要订低脂午餐。先用 LocationTool 获取地址，再用 FoodPlatformTool 检索“沙拉”。
[ACTION] FoodPlatformTool.search(query="salad", address="Current")
[OBSERVE] 发现 A 店已关门。触发反思：切换到 B 店。
[THOUGHT] B 店有满减。准备结算，调用 PaymentTool。

智能化的核心：

不论是获取地理位置、在成千上万个菜品中做语义筛选，还是安全地处理支付，这些都被视为**可被调度的函数**。

Agent 的智能体现为：它不再需要固定的脚本程序（If-Else），而是能够根据**观察到的环境反馈**（例如店关门了、余额不足），动态地调整它对这些 Tool 的调用策略。

"这种模式与 Deep Research 研究文献、Vibe Coding 编写代码在底层架构上是完全一致的。"
