从“聊天框”到“工具箱”

在自主智能体（Autonomous Agents）成为主流前，AI 通过 Function Calling 与 MCP（Model Context Protocol）实现的经典外挂功能形态。

单次交互为主 工具桥接 受控执行

核心机制：能力外挂

大语言模型（LLM）本身是“断网”的概率预测器。通过 Function Calling 或最新的 MCP 协议，模型可以将需求转化为结构化指令，调用外部环境。

User Request → LLM Decision → External Tool

联网智能搜索

形态：ChatBox + Search Engine

当用户询问实时新闻或专业事实时，模型生成搜索关键词，通过 API 获取网页摘要并总结。

"搜索 2024 年巴黎奥运会金牌榜..."
代表案例：Perplexity, ChatGPT Browse

Python 代码解释器

形态：Code Interpreter

在沙箱环境中运行代码，用于复杂数学计算、数据可视化或自动化生成脚本。

"帮我计算这组数据的正态分布并画图..."
代表案例：ChatGPT Advanced Data Analysis

Excel 读写增强

形态：File Connector

从对话中提取关键结构化信息并导出为 .xlsx 文件，或读取用户上传的表格进行清洗和摘要。

"将这段产品对比整理成 Excel 发给我..."
代表案例：Claude Artifacts, Excel Plugins

知识库/PDF 对话

形态：RAG-powered Chat

通过外挂向量数据库，模型可以在数万字的文档中精确检索相关片段，解决上下文窗口限制。

"根据上传的说明书，怎么排查错误..."
代表案例：ChatPDF, Kimi.ai

工作流 API 自动化

形态：Workflow Integrator

通过 Zapier 等中间件，将对话指令转化为发送邮件、创建 Slack 消息或同步 GitHub 等外部操作。

"把这段讨论整理成会议纪要并发邮件给团队..."
代表案例：ChatGPT Plugins (Retired), Zapier AI

图像/视觉外挂分析

形态：Multimodal Add-on

通过 OCR 或图像描述模型外挂，LLM 能够识别截屏中的文字或代码，并进行逻辑调试。

"看这张图，帮我修复里面的报错代码..."
代表案例：GPT-4V, Gemini Vision

演进逻辑：从“工具调用”到“自主思考”

1. 工具性 (Utility)

此时的模型更像是一个掌握了工具箱的客服。它不能改变工作流，只能在用户的指令下（单次交互）执行特定的外挂任务。

2. 环境限制 (Sandbox)

无论是 Python 还是 Excel 处理，通常都运行在高度隔离的环境中。这保证了安全性，但也限制了 AI 直接干预物理系统的能力。

3. 协议标准化 (MCP)

Model Context Protocol (MCP) 的出现，试图让不同 AI 客户端能以通用的方式接入本地文件、数据库和外部服务，是通往 Agent 的关键。
