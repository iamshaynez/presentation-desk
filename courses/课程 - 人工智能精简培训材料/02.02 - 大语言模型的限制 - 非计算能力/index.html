大语言模型的缺陷 - 无计算能力

为什么“最强大脑”连简单的数学题都会算错？

本质原理：是文科生，不是理科生

LLM 的本质是 Next Token Prediction（下一个词预测）。它是在根据概率做“文字接龙”，而不是在运行逻辑代码。
它没有内置计算器。对于它来说，“1+1=2”是因为它读过这句话，而不是因为它进行了运算。

典型翻车现场

大数乘法幻觉：
问：23,894 * 1,203 = ?
LLM 可能会自信地给出一个看似合理但完全错误的数字，因为它在模仿数字的格式，而非计算数值。
字数统计困难（Tokenizer 限制）：
问：单词 "Strawberry" 中有几个 "r"？
许多模型会回答 "2个"，因为在 Tokenizer 眼中，"Strawberry" 可能被拆分为 "Straw" 和 "berry"，它“看”不到字母层面的构成。
