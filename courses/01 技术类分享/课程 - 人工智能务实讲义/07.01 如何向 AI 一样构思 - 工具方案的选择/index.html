四个核心思考维度

知识 vs 事实依赖

Agent 是依赖模型自有的通用常识，还是必须依赖你提供的独有事实数据？

  • ✅ 通用知识：写作、翻译、基础代码、角色扮演
  • ⚠️ 独有事实：公司财报、特定行业Know-how、私人合同

信息源与格式

数据的获取难度和解析难度直接决定了 Agent 的“感知”边界。

网络 (API/搜索)
非结构化 (Doc/Pdf)
结构化 (Excel/SQL)

注：Excel 处理往往需要专门的 Code Interpreter 技能。

任务目标导向

任务属于哪种智力活动？不同目标对应不同的 Prompt 策略。

思考 (逻辑推理/决策) 整理 (提取/分类/摘要) 创作 (生成/续写)

专业工作流

是否需要模拟人类专家的特定步骤？（如：先查法律条文，再做合规对比，最后出报告）

"好的 Agent 不是在直接回答，而是在模拟专业的人类 SOP。"
2

局限性预警：一票否决项

除非拥有可靠的外部技能 (SKILL/Tool Use) 注入，否则以下任务必须谨慎评估：

统计与精确计算

模型在做多位数乘除、复杂统计分析时极不可靠。

✅ 破解方案：使用 Code Interpreter，让 Agent 编写并运行 Python 代码进行数学运算。

事实准确度调研

模型内部知识存在截止日期，对即时事实或微小细节存在幻觉。

✅ 破解方案：强制开启 Google Search 插件或接入可靠的 RAG 向量数据库。

超大上下文依赖

处理数万行文档时，模型会存在“中间遗忘”和召回率下降问题。

✅ 破解方案：采用分段处理、多级提取或知识图谱化，减少单次处理的压力。
3

工具选择逻辑

分类 典型场景 优势 劣势
CLI / API 类 批量数据处理、后端服务集成、自动化脚本 无限可扩展性，可深度对接已有业务逻辑 技术门槛高，缺乏可视化反馈
Copilot 类 沉浸式代码、文档协助、标准 SaaS 套件内增强 开箱即用，功能打磨极度成熟可靠 功能固化，无法为你的业务定制“新技能”
Agent Copilot (如 Coze) 复杂工作流机器人、跨工具调度、私域知识问答 低代码，擅长多步骤、多工具的编排 响应延迟略高，过度依赖平台生态

总结：从“不可行”到“可靠”

评估 Agent 并非在找“它能做什么”，而是在看“它通过什么辅助手段能做对什么”。
