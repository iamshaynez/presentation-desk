大语言模型的缺陷 - 非逻辑性

LLM 本质上是基于统计相关性的“概率预测机”，而非基于符号规则的“逻辑推理机”。在处理严格逻辑任务时，模型常因追求语言流畅性而牺牲事实准确性，产生直觉上合理但逻辑谬误的“幻觉”。

案例一：字面统计幻觉

“Strawberry 有几个字母 r？”

LLM 常见回答

“有两个 r。”
（Token 化导致模型“看不见”字符内部结构）

真实逻辑

3 个。
需要字符级精确计数，而非词汇级概率检索。

案例二：线性诱导谬误

“晾干3件衣服需3小时，晾干30件需多久？”

LLM 常见回答

“30 小时。”
（盲目套用数学线性增长模式）

真实逻辑

3 小时。
物理世界中“晾衣服”通常是并行过程。

案例三：关系传递失效

“Sally 有 3 个兄弟，每个兄弟有 2 个姐妹。”

LLM 常见回答

“6 个姐妹。”
（简单的乘法运算 3x2）

真实逻辑

1 个姐妹（即 Sally 自己）。
需构建一致的人物关系图谱。

核心启示

在涉及严密逻辑推理（如数学证明、代码调试、复杂规划）的场景中，不可盲目信赖 LLM 的直接输出。应采用“思维链（CoT）”引导推理，或让 LLM 生成代码/工具调用来执行逻辑计算。
